### 1. Обзор файловых сервисов

Файловые сервисы предоставляют механизмы для удаленного доступа к файловым системам. Их основная задача — обеспечить прозрачность: пользователь или приложение работает с удаленными файлами так, как если бы они находились на локальном диске.

**Ключевые задачи:**
*   **Централизация данных:** Хранение данных на сервере вместо локальных ПК.
*   **Резервное копирование:** Упрощение бэкапов за счет единой точки хранения.
*   **Контроль доступа:** Разграничение прав на основе пользователей и групп.
*   **Совместная работа:** Одновременный доступ нескольких пользователей к данным.

### 2. Протоколы SCP и SFTP

Оба протокола базируются на инфраструктуре **SSH (Secure Shell)** и предназначены для безопасной передачи данных в сетях Unix/Linux.

#### SCP (Secure Copy Protocol)
*   **Назначение:** Простое копирование файлов между хостами.
*   **Особенности:** Исторически первый протокол передачи файлов поверх SSH. Работает только по принципу «копировать/вставить». Не умеет прерывать сессию с возможностью последующего возобновления (resume), не имеет функций управления файлами (списки, удаление) на удаленном сервере в интерактивном режиме, кроме прямой перезаписи.
*   **Использование:** Команда `scp` в терминале. В настоящее время считается устаревающим в пользу `rsync` или SFTP из-за ограничений по безопасности и функциональности.

#### SFTP (SSH File Transfer Protocol)
*   **Назначение:** Расширенный, безопасный протокол для работы с файлами.
*   **Особенности:** Несмотря на название, не имеет ничего общего с классическим FTP (кроме синтаксиса команд). Работает через одно соединение SSH (порт 22).
*   **Преимущества:**
    *   Поддержка возобновления прерванных загрузок.
    *   Функциональность файлового менеджера (удаление, переименование, изменение прав доступа, создание директорий).
    *   Безопасен по умолчанию (весь трафик шифруется, включая аутентификационные данные).

### 3. Протокол FTP (File Transfer Protocol)

Классический протокол передачи файлов, разработанный в 1971 году.

**Архитектура:**
*   Использует два канала:
    1.  **Управляющий (Control):** Порт 21/tcp. Отправка команд (USER, PASS, GET, PUT).
    2.  **Данных (Data):** Порт 20/tcp (активный режим) или динамические порты (пассивный режим).

**Режимы работы:**
*   **Active:** Сервер сам инициирует соединение к клиенту для передачи данных. Проблематичен при работе через NAT/файрволы.
*   **Passive:** Клиент инициирует оба соединения. Предпочтительный режим для современных сетей.

**Безопасность:**
*   **FTP (Plain):** Передает логины, пароли и данные в открытом виде (base64, не шифрование). Крайне небезопасен для использования в интернете.
*   **FTPS (FTP over SSL/TLS):** Добавляет шифрование через SSL/TLS (обычно порты 989, 990).

### 4. Сервис NFS (Network File System)

Протокол сетевой файловой системы, разработанный Sun Microsystems. Стандарт де-факто для Unix/Linux сред.

**Версии:**
*   **NFSv3:** Самая популярная версия. Работает через UDP/TCP. Требует дополнительных протоколов для аутентификации (RPC, mountd, statd, lockd). Не обладает встроенным шифрованием.
*   **NFSv4:** Современная версия. Упрощает работу (один порт 2049, проход через фаерволы), интегрирует аутентификацию (RPCSEC GSS) и поддержку Kerberos, обеспечивает лучшее блокирование файлов (locking).

**Особенности:**
*   Принцип «монтирования»: удаленная файловая система встраивается в локальное дерево каталогов.
*   Основан на модели *stateless* (NFSv3) — сервер не хранит информацию об открытых файлах, что обеспечивает высокую отказоустойчивость при крашах, но требует аккуратной работы с блокировками.

### 5. Протокол SMB (Server Message Block)

Основной протокол для совместного использования файлов и принтеров в экосистеме Windows. Также известен как CIFS (Common Internet File System).

**История и реализация:**
*   Разработан IBM, позже расширен Microsoft.
*   В мире Linux/Unix реализация протокола осуществляется через пакет **Samba**. Samba позволяет Linux-серверу выступать в роли контроллера домена Windows или файлового сервера, полностью интегрируясь в Active Directory.

**Особенности:**
*   **Операции с файлами:** Поддерживает сложные блокировки (opportunistic locks — oplocks), что критически важно для работы баз данных (например, Microsoft Access, SQL Server) и совместного редактирования документов.
*   **Аутентификация:** Глубокая интеграция с Active Directory (Kerberos, NTLM).
*   **Транспорт:** Использует порты 445/tcp (современный прямой хост) и 139/tcp (NetBIOS — устаревающий).
*   **Версии:** SMB 1.0 (CIFS) — устарел и опасен (использовался для распространения вирусов типа WannaCry). SMB 3.1.1 — современная версия с шифрованием сквозного трафика (AES-128-GCM), поддержкой многоканальности (Multichannel) и прямым доступом к памяти (RDMA).

### Рекомендации по выбору

1.  **Для безопасной передачи единичных файлов в Linux/Unix:** Используйте **SFTP**. SCP стоит избегать в пользу SFTP или `rsync` поверх SSH.
2.  **Для публичных репозиториев (без аутентификации):** Часто используют **FTP** (анонимный), но в современных реалиях безопаснее использовать **HTTPS**.
3.  **Для интеграции Linux-серверов в корпоративную сеть Windows:** Используйте **Samba (SMB)**. Это единственный протокол, обеспечивающий корректную работу с разрешениями NTFS и групповыми политиками.
4.  **Для высокопроизводительных сред с клиентами Linux (HPC, виртуализация):** Используйте **NFSv4**.
